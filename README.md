# ☁️ МетеоБот / Weather Telegram Bot

## 📖 Описание проекта
**МетеоБот** — это удобный и интуитивно понятный Telegram-бот для отслеживания прогноза погоды. Бот управляется кнопочным интерфейсом, но есть возможность обработки ручного запроса с помощью LLM.

## ✨ Основные возможности
* 🌡️ **Текущая погода** — моментальное отображение температуры и реальных ощущений.
* ⚙️ **Гибкие настройки** — быстрая смена города "в один клик".
* 🤖 **Умный UI** — LLM для обработки ручных запросов и динамические клавиатуры, которые меняются в зависимости от контекста диалога.

## 🛠️ Технологический стек
* 🐍 **Язык программирования:** Python (v3.12.10)
* ⚡ **Веб-Фреймворк:** FastAPI (v0.137.1)
* 🤖 **Библиотека бота:** python-telegram-bot (v22.8)
* 🗄️ **База данных:** Redis (v8.8.0) 
* 🌤️ **Погодный API:** OpenWeatherMap / или ваш вариант
* 🐳 **Контейнеризация:** Docker, Docker Compose (v4.78.0)

## 🏗️ Архитектура проекта

* `telegram_bot` — бот-обработчик команд пользователя (управляет диалогом через кнопки). Берет данные через сервер `weather_api`.
* `weather_api` — эндпоинты, бизнес-логика, запросы к Redis-хранилищу.
* `worker` — обращается к API и обновляет данные в Redis-хранилище раз в определенное время. 
* Ollama — LLM(qwen2.5:1.5b), обрабатывающий ручные запросы.
* Redis-лимитёр (slowapi) — ограничение запросов к серверу `weather_api`.
* Redis основное хранилище — хранилище текущих данных о погоде, получает данные от фонового воркера.
* Redis запасное хранилище — хранилище на случай отсутствия ответа от API, воркер получает данные от него. В случае доступа к API воркер дублирует данные в это хранилище.

## 🚀 Быстрый запуск
Приложение **полностью** контейнеризовано, для запуска на Windows необходим `Docker Desktop`.

### 1. Клонирование репозитория
```bash
git clone https://github.com/danilpashin/weather-chatbot.git
cd weather-chatbot
```

### 2. Настройка окружения
Создайте `.env` файл в корне проекта, скопировав `.env.example` файл. Вставьте свои токены API и Telegram-бота:
```bash
cp .env.example .env
```

### 3. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 4. Установка LLM и запуск контейнеров
```bash
docker exec -it ollama_service ollama run qwen2.5:1.5b
docker compose -f docker-compose.yml up
```

### 5. Использование бота
Бот запускается через команду `/start`
